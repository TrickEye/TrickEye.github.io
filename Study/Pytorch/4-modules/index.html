    Pytorch神经网络模型

    这里主要介绍的是 torch.nn 包内提供的各种模型啦。虽然神经网络的概念听起来弯弯绕绕的，但是 Pytorch 实在是太傻瓜式啦，直接给你提供了全面的模型给你来使用，让人很疑惑这么牛的软件居然不收费。

    Convolution 卷积⚓︎

    一个大的矩阵叫做input，一个尺寸比他小的矩阵叫做卷积核，卷积核从input最左上方出发，对应的output的最左上方的值等于卷积核和左上角各对应位的乘积之和。算完output的这一位后，卷积核向右移动算下一位。

    操作很复杂，但是pytorch实现不是很难。

    origin = torch.tensor([[1,2,0,3,1],
                       [0,1,2,3,1],
                       [1,2,1,0,0],
                       [5,2,3,1,1],
                       [2,1,0,1,1]], dtype=torch.float64, device=device)
kernel = torch.tensor([[1,2,1],
                       [0,1,0],
                       [2,1,0]], dtype=torch.float64, device=device)
print(origin.shape)
print(kernel.shape)

    torch.Size([5, 5])

    torch.Size([3, 3])

    接下来我们需要改变两个张量的形状，因为Pytorch的二维卷积API需要至少3维（unbatched），至多4维（batched）张量。 

    origin = torch.reshape(origin, (1, 1, 5, 5))
kernel = torch.reshape(kernel, (1, 1, 3, 3))

    随后我们就可以使用卷积操作了 

    output = F.conv2d(origin, kernel, stride=1)
output2 = F.conv2d(origin, kernel, stride=2)

    这里stride关键字用于指出卷积核位移的长度，也就是说，卷积核不一定仅仅移动一位。此外，还有padding关键字用来指出是否需要在计算时在矩阵周围填充。

    下面我们来试一试通过使用Pytorch提供的模板来实现神经网络中的卷积层

    class Model(nn.Module):
    def __init__(self):
        super(Model, self).__init__()
        self.conv1 = Conv2d(in_channels=3, out_channels=6, kernel_size=3, stride=1, padding=0)

    def forward(self, x):
        output = self.conv1(x)
        return output

    首先我们需要定义一个自己的类，这个类继承了torch.nn.Module类，并且需要实现__init__(), forward()两个方法。__init__()方法创建了这个模型，然后指出了一个卷积层变量。这个变量将会对传入的变量实施一个二维卷积运算，在创建这个卷积层的时候，你需要指出接收的channel，输出的channel，卷积核的大小，以及stride和padding。

    forward方法是每一个继承自nn.Module的类都应该重写的方法，他指出了这个模型的一次前向运算所经历的过程。在这个例子里，他对传入的x张量使用模型内的conv1运算来卷积，输出得到的结果。Pytorch有一点很聪明，你看这里conv1虽然是一个变量，但是可以直接调用，而后面，我们可以看到，我们实例化的Model示例也是可以被调用的。再或者，我们可以说，Conv2d是nn.Module的一个子类，放在我们的Model里面，充当了一个子模型。

    那么我们怎么用这样一个自定义子类呢？当然是实例化他啦！

    我们注意到，我们只是为模型提供了卷积核的尺寸，但是卷积核的具体取值是不知道的。在实际计算过程中，Pytorch会预先填充卷积核，通过梯度下降来优化卷积核的参数。这里卷积核的具体取值就是我们神经网络要训练的参数之一啦。卷积层还有另外的偏置量也是可以训练的。

    Pooling 池化⚓︎

    仍然是一个input矩阵和一个池化核kernel，池化运算是在池化核对应的区域内找出一个符合特征的值来作为结果矩阵的对应结果，这个符合特征的特征就区分了不同的池化操作，包括最大池化，最小池化，平均池化。

    池化的参数还包括：

    === note "" stride : 步长，注意这里默认值是kernel_size，同conv有变化

    `padding`
:   填充0

`ceil_mode`
:   用于计算输入矩阵尺寸的时候，是采取地板还是天花板取证

    示例 

    origin = torch.tensor([[1,2,0,3,1],
                       [0,1,2,3,1],
                       [1,2,1,0,0],
                       [5,2,3,1,1],
                       [2,1,0,1,1]], dtype=torch.float64, device=device)
origin = torch.reshape(origin, (-1, 1, 5, 5))

from torch.nn import MaxPool2d


class Pooling(nn.Module):
    def __init__(self):
        super(Pooling, self).__init__()
        self.maxpool1 = MaxPool2d(kernel_size=3, stride=1, ceil_mode=True)

    def forward(self, x):
        output = self.maxpool1(x)
        return output

pooling = Pooling()
out = pooling(origin)

    MaxPooling模型没有可供训练的参数，他存在的意义主要是为了大幅缩减数据量，加快训练速度。

    其他模型，组装，序列化⚓︎

    Pytorch.nn提供了很多其他的模型，包括BatchNorm正则化层，线性层，非线性激活函数等等。

    我们说过nn.Module是一个具有相当高的拓展性的模块，我们能看到的很多神经网络也绝非一两层就能解决的事情。这也就意味着需要在torch提供的诸多模型之间选择合适的模型和数量，采用某种办法组装，组合成一个大的模型，对这个大模型加以训练。

    我们还是可以通过继承nn.Module，修改他的子类的初始化和forward方法来实现自己的模型

    class Model(nn.Module):
    def __init__(self):
        super(Model, self).__init__()
        self.conv1 = Conv2d(in_channels=3, out_channels=32, kernel_size=5, padding=2)
        self.maxPool1 = MaxPool2d(kernel_size=2)
        self.conv2 = Conv2d(in_channels=32, out_channels=32, kernel_size=5, padding=2)
        self.maxPool2 = MaxPool2d(kernel_size=2)
        self.conv3 = Conv2d(in_channels=32, out_channels=64, kernel_size=5, padding=2)
        self.maxPool3 = MaxPool2d(kernel_size=2)
        self.flatten = Flatten()
        self.linear1 = Linear(in_features=1024, out_features=64)
        self.linear2 = Linear(in_features=64, out_features=10)

    def forward(self, x):
        x = self.conv1(x)
        x = self.maxPool1(x)
        x = self.conv2(x)
        x = self.maxPool2(x)
        x = self.conv3(x)
        x = self.maxPool3(x)
        x = self.flatten(x)
        x = self.linear1(x)
        x = self.linear2(x)
        return x

    另一种方式是通过序列化，Sequential

    from torch.nn import Sequential


class Model1(nn.Module):
    def __init__(self):
        super(Model1, self).__init__()
        self.sequence = Sequential(
            Conv2d(in_channels=3, out_channels=32, kernel_size=5, padding=2),
            MaxPool2d(kernel_size=2),
            Conv2d(in_channels=32, out_channels=32, kernel_size=5, padding=2),
            MaxPool2d(kernel_size=2),
            Conv2d(in_channels=32, out_channels=64, kernel_size=5, padding=2),
            MaxPool2d(kernel_size=2),
            Flatten(),
            Linear(in_features=1024, out_features=64),
            Linear(in_features=64, out_features=10)
        )

    def forward(self, x):
        x = self.sequence(x)
        return x

    序列化提供了一种更简便的写法，将所有计算过程放在了一个序列里面，再将整个计算过程整理成这个序列从输入到输出的过程。

    模型保存，读取⚓︎

    # 保存网络的结构和参数，体积略大
torch.save(model_to_be_saved, "path/of/model.any_extend")
model = torch.load("path/of/model.any_extend")

# 网络的结构每次创建的时候都可以以相同形式组织，
# 重要的是参数，因此下面的方法只保存参数，体积会小一点。
torch.save(model_to_be_saved.state_dict(), 
           "path/of/model.any_extend")
model = Model()
model.load_state_dict("path/of/model.any_extend")
